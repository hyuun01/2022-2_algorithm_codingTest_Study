구현 문제
: 풀이를 떠올리는 것은 쉽지만 소스코드로 옮기기 어려운 문제

ex) 
알고리즘은 간단한데 코드가 지나칠 만큼 길어지는 문제,
특정 소수점 자리까지 출력해야 하는 문제,
문자열이 입력으로 주어졌을 때 한 문자 단위로 끊어서 리스트에 넣어야 하는 문제

- brute-force(완전탐색)
- simulation 
    : 문제에서 제시한 알고리즘을 한 단계씩 직접 수행해야
    : 특정 알고리즘이나 자료구조에 종속되지 않고 문제 상황에 대해 구현해야

구현 시 고려해야 할 메모리 제약 사항
1. C/C++에서 변수의 표현 범위
int 4바이트, longlong 8바이트와 같이 자료형 크기가 정해짐
BingInteger 클래스로 가변적인 정수형 자료를 표현 가능

2. 파이썬에서 리스트 크기
파이썬에서는 프로그래머가 변수 자료형 지정 안함
B 실제 저장될 때는 그 저장되는 자료형만큼 공간 차지
-> 리스트 크기가 너무 큰 경우에는 메모리 공간 초과할 수도 있음에 주의!
ex) 숫자 5 -> 시스템이 int 자료형으로 저장 -> 4byte만큼의 공간 차지

3. 채점환경
파이썬 코드 : "1초에 2000만번의 연산 수행" 한다고 보통 가정
C/C++ 코드 : "파이썬 코드의 절반 정도 시간" 이라고 보통 가정
